## Problema real

| Prueba | Input | max_tokens | Resultado |

| Contexto grande | ~5k tok | 4000 | ✅ |
| Solo architecture | ~1.8k tok | 8192 | ✅ |
| Solo contracts | ~5.5k tok | 8192 | ✅ |
| **Contexto completo, output corto** | **~8k tok** | **500** | **✅** |
| **Contexto completo, output largo** | **~8k tok** | **8192** | **❌ disconnect** |

## Traducción simple
- el input pasa bien
- Sonnet empieza a generar
- pero la generación de ~8k output tokens toma demasiado tiempo
- y **la conexión se cae** antes de terminar

## Causa raíz
No hay `request_timeout` explícito en tu config de LiteLLM:
- `/home/robotin/.config/litellm/config.yaml`

Sin timeout explícito, LiteLLM usa un default que no alcanza para generaciones grandes de Anthropic.

## Fix recomendado
Agregar a tu config de LiteLLM:

```yaml
litellm_settings:
  request_timeout: 600
```

Y reiniciar LiteLLM.
