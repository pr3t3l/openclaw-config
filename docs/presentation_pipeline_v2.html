Declassified Cases Pipeline

10-PHASE AI CASE GENERATION · CONCEPT → PACKAGED PDFs
Declassified Cases are mystery detective games — sealed evidence envelopes containing police files, forensic reports, witness interviews, and other documents. Players solve fictional crimes using the clues embedded across 15-25 documents. This pipeline is the AI system that generates these cases from a one-line concept to packaged, store-ready PDFs. No templates — every document is unique.
01The 10 Phases: Concept to Product

A case starts as a concept — a setting, a victim, a crime — and moves through 10 phases until it becomes a ZIP file containing PDF envelopes ready for sale at declassified.shop. Each phase is handled by a specialized AI skill or script. The entire process costs $6-8 per case, with the rendering phase (where Claude generates the visual HTML for each document) accounting for most of the cost.

02The Zero-Templates Revolution

Before V9, the pipeline used Handlebars templates — pre-built HTML skeletons where content was inserted into fixed slots. This worked for simple documents but broke down fast. A police interrogation transcript needs a completely different layout than a social media screenshot or a newspaper front page. Templates were too rigid. When the AI wanted to create a floor plan diagram or a surveillance camera timeline, the templates couldn't handle it.

V9 eliminated templates entirely. Now Claude receives the content (a _content.md file with text and [IMAGE:] placement tags) plus a design brief, and generates complete standalone HTML from scratch — CSS, layout, typography, aging effects, stamps, everything. Then headless Chromium converts it to PDF. Every document is unique. A forensic toxicology report looks different from an Instagram DM screenshot, which looks different from a handwritten note.

"Quality is designed in the clue catalog, not QA'd afterwards."

The pipeline uses a two-tier prompt system for rendering: text-heavy documents (transcripts, court orders, witness statements) get a minimal styling prompt, while visual documents (social media mockups, floor plans, POI sheets) get a full design system prompt with detailed visual instructions. One strict rule applies to all documents: backgrounds must always be white or light — these are meant to look like real paper documents, not dark-themed web pages.

03The Spawn Problem (and Its Solution)

OpenClaw's built-in sub-agent spawner (sessions_spawn) is broken for file-writing tasks. Sub-agents complete with "(no output)" and zero tokens — the work vanishes. This was confirmed broken 6+ times across multiple cases and phases. It's an upstream platform limitation that can't be fixed.

The replacement: three spawn scripts that bypass OpenClaw's spawner entirely. Each script calls the AI API directly via streaming curl (Python's requests library fails in WSL for long API calls — a hard-learned lesson), and the script writes the output files, not the sub-agent. This means cost tracking works, file writing is reliable, and timeouts are controllable.

spawn_agent.py
Generic content generation. Used for Art Director, Experience Designer, Production Engine, and QA phases. Calls the API, receives the response, writes files to disk.
spawn_narrative.py
Specialized for the Narrative Architect. Supports --phase plan (case plan) and --phase catalog --envelope X (per-envelope clue catalog generation).
spawn_images.py
POI portrait generation via DALL-E 3 or Gemini's nano-banana model. Downloads images with wget (not curl — another lesson learned).
04Quality: 6 Pillars, 60 Points

Every case is scored against a 6-pillar quality framework totaling 60 points. The framework was developed by benchmarking against competitors (Hunt A Killer, mododetective.store) and an internal gold standard case. The key insight: User Experience / Emotional Arc gets 2x weight — a case with perfect clue structure but no emotional payoff is a bad product.

PillarWeightMaxWhat it measures
UX / Emotional Arc2x20"Aha!" moments, suspense, immersion, conflicting empathy, catharsis
Information Relevance1x10Every clue is purposeful — no filler documents
Clue Structure1x10Cognitive load management, progressive revelation across envelopes
Visual Support1x10Document design enhances the content (not just decorates it)
Dynamic Variety1x5Document types are varied — not 15 interrogation transcripts
Document as Experience1x5Each document has personality — it feels like a real artifact

The pipeline targets a specific emotional journey: intellectual "aha!" satisfaction, suspense and tension, immersion and flow, conflicting empathy vs suspicion toward suspects, the pleasure of not knowing, smugness when deducing early, catharsis at resolution, frustration from well-placed red herrings, and camaraderie in group solving. QA is 87% deterministic (scripts check structure, completeness, clue coverage) — the AI is only used for narrative judgment calls.

05Case Tiers & Document Types

Cases come in three tiers — SHORT, NORMAL, and PREMIUM — each with different constraints on complexity, document count, and production cost. The pipeline currently produces NORMAL tier cases. PREMIUM tier includes audio narration via ElevenLabs TTS (designed but untested).

DimensionSHORTNORMALPREMIUM
Suspects4-56-88-12
Documents10-1515-2222-30
Pages15-2525-3838-55
Envelopes2-33-44-5
Social media mockups1-23-55-8
Cross-envelope links2-34-66-10

Only 2 documents are mandatory in every case: a Case Introduction (always first in Envelope A) and a POI Sheet (always second). All other documents are chosen by the Narrative Architect based on the story. The system has an extensible document type catalog — interrogation transcripts, official memos, newspaper articles, social media screenshots, floor plans, forensic reports, court orders, and many more. Custom types use a design_hint field to guide the renderer.

Cases also follow one of two experiential styles: físico/local (handwritten notes, receipts, napkin messages — intimate, tactile feeling) or digital/corporativo (social media posts, Slack messages, app screenshots — modern, tech-savvy feeling). The Narrative Architect chooses the style based on the story's setting and characters.

06The Evolution: V5 → V9

The pipeline didn't start at V9. It took 9 iterations to reach the current architecture, and each version taught something critical. The journey from V5 to V9 is why the Meta-Workflow Planner was eventually built — to avoid needing 9 versions for every future workflow.

09Key Decisions & Lessons
#DecisionWhy
1Zero templates (V9)Templates too rigid — Claude can generate any document type from scratch
2Quality through structureQuality is designed in the clue catalog, not QA'd afterwards
3Images AFTER QADALL-E costs wasted if content rejected — generate only after approval
4Hybrid QA (87% deterministic)Scripts check structure; LLM only for narrative judgment
5Model abstractionSKILL.md references reasoning levels, not model names — one config to update
6Smart retry (3 max)Each retry must diagnose the failure. Never blind retry. After 3 → stop + alert
7Envelope-by-envelope generationFull clue catalog too large for single spawn (~50KB crashes). Generate in chunks.
8Lessons-learned agentEach case improves by learning past failures from structured JSON
9Dynamic document selectionOnly 2 mandatory docs. Narrative Architect chooses the rest per story.
10Skills not agentsPipeline "agents" are skills in the workspace, not separate OpenClaw agents
LESSONS: Python requests DIES in WSL for long API calls — always streaming curl via subprocess. subprocess timeout must exceed curl --max-time (650 > 600). NEVER dark backgrounds in rendered HTML — documents are paper. Sonnet truncates JSON above ~8K tokens — generate by blocks. SKILL.md needs concrete JSON skeleton examples, not just field lists. Never patch failed output with scripts — delete and re-run with augmented prompt.
