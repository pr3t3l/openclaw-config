OpenClaw Platform & Robotin

INFRASTRUCTURE · CEO AGENT · MEMORY · MODEL ROUTING
The foundation that runs everything else. Three services (a database, an AI model proxy, and a gateway), 24 AI models, 3 Telegram bots, and a CEO agent named Robotin that orchestrates daily operations. This is the infrastructure layer — if it goes down, nothing works. If it runs well, you forget it exists.
01Three Services That Start Themselves

Everything runs on WSL Ubuntu on a Windows machine. When the machine boots, a startup shortcut triggers WSL, which runs a boot hook that calls a master startup script. That script starts three services in order — database first, then the AI model proxy, then the gateway that connects everything to Telegram. The script is idempotent: running it twice doesn't break anything, and it checks whether each service is already running before attempting to start it.

LiteLLM is the model proxy — a single endpoint that routes API calls to 24 different AI models from OpenAI, Anthropic, Google, and OpenRouter. When a skill needs "a thinking-capable model," it calls LiteLLM at port 4000, and LiteLLM routes to the right provider with the right credentials. This means model changes happen in one config file, not across dozens of scripts.

BOOT CHAIN: Windows boot → WSL.lnk (in shell:startup) → WSL starts → wsl.conf [boot] command → start_all_services.sh → PostgreSQL → LiteLLM → OpenClaw Gateway → 3 Telegram bots come online.
02From $43/week to ~$0 for GPT

In early March 2026, the system was spending $43.36 per week on API calls — 96% of it on GPT-5.2 medium, which all three agents used as their primary model. That's $173/month just on language model tokens.

The fix: Codex OAuth. OpenAI's Pro subscription ($200/month) includes unlimited API access through a device-code OAuth flow. All three agents now use openai-codex/gpt-5.4 as their primary model at $0/token. Pipeline skills use the same models through LiteLLM. The only remaining variable cost is Claude Sonnet for document rendering in the Pipeline (~$2 per case, paid directly to Anthropic).

The $200/month Pro subscription replaced $173/month in per-token costs — and upgraded all agents from GPT-5.2 to GPT-5.4 in the process.
IMPORTANT: GPT-5.4-pro (thinking model) is NOT available on the Pro subscription via Codex. Only the standard GPT-5.4. Claude handles rendering. Gemini handles some image generation.
03Robotin: The CEO Agent

Robotin is the personal AI assistant and system orchestrator. It handles daily tasks (questions, lookups, writing), system administration (service health, model availability, API spend monitoring), finance tracking (the Finance Tracker is a skill inside Robotin), and routes complex tasks to specialized agents. It lives in Telegram at @Robotin1620_Bot.

Every session starts the same way: Robotin reads its personality file (SOUL.md: "Be genuinely helpful, not performatively helpful"), the user profile, today's and yesterday's daily notes for recent context, and — in main sessions — its long-term memory file. This gives it continuity across conversations.

What Robotin Does
Answers questions, manages system health, routes "Planifica:" messages to the Workflow Creator bot, sends budget alerts and payment reminders (via Finance Tracker skill), monitors API costs, and handles marketing operations.
What It Doesn't Do
Robotin never modifies the Declassified workspace directly — workspace isolation is enforced. It never uses mini/nano models for content or QA tasks (quality floor). And it never auto-approves decisions without human confirmation.

05The 3-Layer Memory System

AI agents forget everything between conversations. The memory system gives Robotin continuity — it remembers what happened yesterday, what decisions were made last month, and can search across its entire history.

Layer 1: Daily Notes
Files named by date (memory/YYYY-MM-DD.md) — raw logs of what happened each day. Today's and yesterday's notes are loaded automatically at the start of every session. Written by the agent during conversations.
Layer 2: Long-Term Memory
A curated file (MEMORY.md) containing decisions, preferences, key facts, and system state. Loaded only in main sessions — not in sub-agent or heartbeat sessions. Manually maintained.
Layer 3: Semantic Search (QMD)
QMD 2.0.1 + Bun runtime — semantic search across all memory files. Updates every 5 minutes, returns up to 6 results within a 4-second timeout. Only available in direct chats, not sub-agent contexts.

When the agent's context window gets full, a compaction process kicks in. But before any context is pruned, a memoryFlush at 6,000 tokens saves the current conversation state to daily notes. This prevents the agent from "forgetting" what it was doing mid-task.

06Model Routing: Two Permission Layers

Adding a new AI model to the system requires updating two places in the configuration — and forgetting either one means the agent can't use it. The first layer (models.providers.litellm.models) declares which models exist and what capabilities they have. The second layer (agents.defaults.models) authorizes specific models for agent use.

This two-layer design is intentional — it prevents agents from accidentally using expensive or inappropriate models. But it's also a common source of confusion: if you add a model to LiteLLM but forget to authorize it in openclaw.json, the agent will fail silently.

07Remote Access

The system is accessible remotely via Tailscale — a mesh VPN that connects devices without port forwarding. SSH from phone via Termius to the WSL instance, then use tmux sessions for persistent work that survives disconnections. Claude Code can be dispatched remotely through these sessions, making it possible to manage the entire system from a phone while traveling.

09Key Decisions & Lessons
#DecisionWhy
1Two permission layers for modelsPrevents accidental use of expensive models. Both must be updated together.
2Codex OAuth for $0/token GPTEliminated $43/week in API costs
3Tailscale (not port forwarding)OpenClaw rejects bind:0.0.0.0. Tailscale is native integration
4No master_key on LiteLLMBreaks OpenClaw auth. Dashboard runs without auth (localhost-only)
53-layer memoryEach layer serves a different retention need: daily, curated, searchable
6compaction: safeguard + memoryFlushAgents save context before compaction — never lose mid-task state
7WSL.lnk + wsl.conf boot hookMore reliable than .bat keepalive. Native WSL auto-start.
8CEO workspace isolationNEVER modify workspace-declassified/ from CEO context

LESSONS: OpenClaw doesn't resolve ${ENV_VARS} in apiKey — use literal values. echo >> for .env creates duplicates — edit with nano. LiteLLM Prisma gives duplicate model warnings (not blocking, just noisy). Gemini Flash hallucinated model names in a boot loop, burning credits (always verify model names). When LiteLLM can't bind to its port, it silently falls back to a random high port.
